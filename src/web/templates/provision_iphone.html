📱 iPhone aprs.fi Setup Steps
1
Start the BLE KISS bridge
The kp4pra-tnc-ble service should already be running and advertising. Check the status card above. If stopped, click "Restart BLE Bridge".
2
Confirm Bluetooth is enabled on iPhone
Open iOS Settings → Bluetooth. Bluetooth must be On. Do NOT pair KP4PRA TNC from the iOS Settings Bluetooth list. aprs.fi connects from within the app, not from iOS Settings.
3
Open aprs.fi on iPhone
Tap the TNC button or go to Settings → TNC Connection. Select Bluetooth Low Energy KISS as the connection type.
4
Select KP4PRA TNC from the device list
aprs.fi will scan for BLE KISS devices and show KP4PRA TNC in the list. Tap it to connect.
5
Verify KISS data is flowing
The BLE Client status above should show "Connected". APRS packets received by Dire Wolf will appear in aprs.fi.
ℹ Important Notes
  • Only one BLE client can connect at a time.
  • If the iPhone disconnects, the bridge waits for a new connection automatically.
  • If Dire Wolf restarts, the BLE bridge reconnects automatically.
  • BLE connect/disconnect events do not write to disk.
  • BLE connect/disconnect does not require a reboot.
  • To change the BLE device name permanently, use the Configuration page. This requires a reboot.
  • Raw KISS binary data passes through unchanged — no APRS decoding, no traffic saved.
